Our position

Science that Compounds: the Need for a New Substrate for Research in the Age of AI

Lanusse & Parker · May 2026

In this position paper, published last month, we imagine this future of accelerated research, where every scientist works with AI assistance.

Our outlook remains resolutely optimistic: this future can unleash curiosity, open up far more ambitious questions, and multiply the pace at which results come to light.

Yet that very speed makes each result harder to check: a gap is widening between what is produced and what can be verified.

From there, none of these benefits will materialize if we can no longer tell what is trustworthy — what can be built upon — from what is not.

… and this need for trust arises at every stage of a result’s life:

When the scientist reviews what their agent just produced.

When a reviewer must judge an analysis they did not carry out themselves.

When a third party later tries to reproduce or extend the work.

A new substrate for scientific results

It all starts with the form a scientific result takes.

Code

Executable but opaque.
Buried assumptions, missing intent.

The missing link

Decisions, assumptions,
evidence, provenance

Paper

The traditional format.
Readable but lossy — the analysis cannot be regenerated.

A result can be efficiently assessed — by a human or by a machine — when three conditions are met:

Certified provenance

Every figure, number and claim traces back to the data, code and decisions that produced it — without ever requiring re-execution.

Fully observable

The code and artifacts, but also every decision — estimator, prior, threshold, dataset — and the reasoning behind it.

Scientifically legible

Organized around the claims, decisions and ideas that matter, with direct access to the evidence and code behind every point.

Every result becomes a solid foundation for the next — science that compounds at scale, robust to the acceleration of scientific production.

Lightcone‑CLI · agent layer

The available skills

Every project starts with a set of Claude Code skills. You drive the agent with /lc-new, /lc-from-code, /lc-from-paper

Entry points — depending on your starting point

/lc-new — from a research question

Interactive scoping: surfaces the decisions, searches the literature, extracts verified citations as prior insights, drafts universes. No YAML written by hand.

/lc-from-code — from an existing codebase

Scans the repository, drafts astra.yaml, parameterizes the scripts so decisions can vary — the existing logic stays intact.

/lc-from-paper — reproduce a paper

ORIENT → ralph-loop reproduction: extracts the paper, interviews you, clones the reference code, then iterates ARCHITECT → SPECIFY → LITERATURE → IMPLEMENT → RUN → COMPARE under a per-paper constitution.

/lc-feedback — report a bug

Opens a GitHub issue with the version & session context attached automatically.

damping_prior_gaussian_fiducial open paper →

Tight Gaussian priors (width 1–2 h−1 Mpc) on the BAO damping parameters centred on theory values are recommended; fully fixing them risks biases and flat priors can also bias α and weaken constraints on noisy data.

damping_prior_width_justification

Biases of order 0.1% in αiso and 0.2% in αAP appear when the damping parameters are misspecified by more than ∼2 h−1 Mpc from their fiducial values — justifying Gaussian priors of width 1–2 h−1 Mpc.

sigma_perp_parallel_formula

Σ⊥ and Σ∥ follow Σ⊥ = Σ0·G, Σ∥ = Σ0·G·(1+f), with Σ0 = 12.4 h−1 Mpc for σ8 = 0.9 at z = 0 — the theoretical prediction that anchors the Gaussian prior centres.

Alternative

flat

damping_prior_gaussian_fiducial

Flat priors can bias α and weaken constraints on noisy data; the same Chen+ 2024 analysis that motivates the Gaussian fiducial also rules flat priors out as the headline choice.

bias_weak_effect_on_sigma

Because Lagrangian displacements are dominated by bulk flows, galaxy bias has only a small effect on Σ⊥/Σ∥. With a flat prior the fit has no anchor against this near-degenerate direction.
